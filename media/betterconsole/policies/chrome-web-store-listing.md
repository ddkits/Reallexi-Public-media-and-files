## Short Description

Capture, group, explain, search, and export browser console issues locally.

## Single Purpose

betterConsole improves browser console debugging by capturing console activity locally, grouping repeated issues, helping developers search and understand errors, generating sanitized debug reports, and opening user-controlled fix searches.

## Detailed Description

betterConsole is a privacy-first debugging extension for Chrome, Edge, Brave, and Chromium-based browsers. It captures console logs, warnings, errors, runtime exceptions, unhandled promise rejections, failed resource loads, and CSP violations without breaking native console behavior.

The extension groups repeated errors, highlights likely source files, detects framework hints, provides practical fix suggestions, and gives developers a Vision Mode overview of the current page's console health. It follows the active tab by default, refreshes capture on tab switches and page loads, and offers an explicit all-tabs mode when developers need a wider debugging view. Clicking an issue opens an inline accordion with root-cause details, source/page context, solution steps, research links, notes, report, and stack trace. Developers can pause or resume capture, refresh the active tab, clear current-tab logs, reset all logs, mark issues fixed, ignore noisy issues, add notes, copy clean error text, export issues, and generate sanitized Markdown debug reports for GitHub Issues, Jira, Linear, Slack, Discord, email, and pull request comments.

betterConsole stores data locally in browser storage. It does not send logs to a server, does not collect analytics, and does not execute remote code. Online fix search is user-controlled: the extension generates a sanitized query, shows it for review, and opens a normal browser search tab only after the user chooses to search. Chrome opens Google search, and Edge opens Bing search in the same browser.

## Permission Justifications

- `storage`: Local logs, settings, notes, bookmarks, ignored patterns, capture state, startup keep/reset preference, and retention state.
- `activeTab`: User-triggered capture refresh for the active tab.
- `tabs`: Active tab metadata, tab switch refresh, tab-scoped counts, side panel opening, current-tab clearing, and user-requested solution search tabs.
- `sidePanel`: Side panel debugging workflow.
- `alarms`: Local retention cleanup.
- `downloads`: User-requested issue/report exports.
- `scripting`: User-triggered and active-tab reinjection of the packaged capture script.
- `unlimitedStorage`: Larger local debugging sessions without server transmission.
- `<all_urls>`: Required for content-script capture on pages developers choose to debug.

## Privacy Practices

- Remote code: No.
- Analytics: No.
- Data sale or transfer: No.
- Data transmitted to developer servers: No.
- User-controlled exports/searches: Yes.
- Locally stored data: Console messages, errors, stack traces, page URLs, tab IDs, timestamps, settings, issue notes, and report content.
